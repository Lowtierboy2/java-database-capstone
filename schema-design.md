# Schema Design

## Section 1: MySQL Database Design (Relational Data)
The Smart Clinic system stores structured relational data in MySQL using Spring Data JPA.
The relational schema includes four core tables: `patients`, `doctors`, `appointments`, and `admin`.
Because doctor availability is modeled with `@ElementCollection`, Hibernate also creates a supporting table for doctor time slots.

### Patients Table
| Column Name   | Type         | Constraints |
|--------------|--------------|-------------|
| patient_id   | BIGINT       | PRIMARY KEY, AUTO_INCREMENT |
| first_name   | VARCHAR(50)  | NOT NULL |
| last_name    | VARCHAR(50)  | NOT NULL |
| email        | VARCHAR(191) | NOT NULL, UNIQUE |
| phone        | VARCHAR(255) | NOT NULL |
| date_of_birth| DATE         | NOT NULL |
| password     | VARCHAR(255) | NULL |
| created_at   | DATETIME     | GENERATED BY HIBERNATE |

### Doctors Table
| Column Name   | Type         | Constraints |
|--------------|--------------|-------------|
| doctor_id    | BIGINT       | PRIMARY KEY, AUTO_INCREMENT |
| first_name   | VARCHAR(50)  | NOT NULL |
| last_name    | VARCHAR(50)  | NOT NULL |
| specialty    | VARCHAR(100) | NOT NULL |
| email        | VARCHAR(191) | NOT NULL, UNIQUE |
| phone        | VARCHAR(255) | NOT NULL |
| password     | VARCHAR(255) | NULL |

### Doctor Available Times Table
| Column Name   | Type         | Constraints |
|--------------|--------------|-------------|
| doctor_id    | BIGINT       | FOREIGN KEY → doctors(doctor_id), NOT NULL |
| available_times | VARCHAR(255) | NULL |

### Appointments Table
| Column Name      | Type         | Constraints |
|-----------------|--------------|-------------|
| appointment_id   | BIGINT       | PRIMARY KEY, AUTO_INCREMENT |
| patient_id       | BIGINT       | FOREIGN KEY → patients(patient_id), NOT NULL |
| doctor_id        | BIGINT       | FOREIGN KEY → doctors(doctor_id), NOT NULL |
| appointment_time | DATETIME     | NOT NULL |
| status           | VARCHAR(255) | DEFAULT 'SCHEDULED' |
| notes            | TEXT         | NULL |

### Admin Table
| Column Name   | Type         | Constraints |
|--------------|--------------|-------------|
| admin_id      | BIGINT       | PRIMARY KEY, AUTO_INCREMENT |
| username      | VARCHAR(50)  | NOT NULL, UNIQUE |
| password_hash | VARCHAR(255) | NOT NULL |
| role          | VARCHAR(255) | DEFAULT 'ADMIN' |

## Section 2: MongoDB Collection Design (Document Data)
MongoDB stores flexible document-based prescription data in the `prescriptions` collection.
This matches the current Spring Data MongoDB model used by the application.

```json
{
  "_id": "67a12f9c8b23d90145ef12aa",
  "patientName": "John Doe",
  "medication": "Amoxicillin",
  "appointmentId": 12,
  "dosage": "500mg twice daily for 7 days",
  "doctorNotes": "Take after meals and return if symptoms persist.",
  "instructions": [
    "Take with water",
    "Avoid skipping doses",
    "Complete the full course"
  ]
}
```

This design reflects the current codebase contract used by the `Prescription` model and repository.
